# 决定下一步做什么

为了改进算法，需要考虑下一步需要做什么。

> [!NOTE] Note
> **高偏差 high bias**: 欠拟合，模型在训练集表现较差
   **高方差 high variance**: 过拟合，模型在测试集表现较差

修正高偏差
1. 增加特征的数量。
2. 增加多项式特征。
3. 减少正则化程度 ($\lambda$)。

修正高方差：
1. 获得更多的训练样本。通常是有效的，但是代价较大。
2. 减少特征的数量。
3. 增加正则化程度 ($\lambda$)。

# 评估假设

为了检验算法，通常需要在**随机排序**后，再分别选取训练集和测试集 (7:3)，以保证均含有各种类型的数据。

使用测试集来评估训练集学习出的参数时，有两种方式计算误差：
1. 线性回归，用测试集计算代价函数 $J$
2. 逻辑回归，除了计算代价函数外，还可以定义误分类的比率。在所有测试集的结果上取平均。
$$
err(h_\theta(x),y)=
\begin{cases} 1 \ \ if \ \ h(x)\ge 0.5 \ \ and \ \ y=0 \ \ or \ \ if \ \ h(x)<0.5 \ \ and \ \ y=1 \\
0 \ \ otherwise 
\end{cases} 
$$

# 模型选择和交叉验证集

讲数据分为 3 个样本集：训练集 (60%)、交叉验证集 (20%)和测试集 (20%)。

> [!example] 10 个不同次数的二项式模型
> 虽然高阶的多项式更能适应训练集，但是泛化不一定好。
> $$
 \begin{align}
 f_{\vec{w},b}(\vec{X})&=w_1x_1 \\
 f_{\vec{w},b}(\vec{X})&=w_1x_1 + w_2x^2 + b \\
 f_{\vec{w},b}(\vec{X})&=w_1x_1 + w_2x^2 + w_3x^3 + b \\
 \vdots \\
 f_{\vec{w},b}(\vec{X})&=w_1x_1 + w_2x^2 + \dots + w_{10}x^{10} + b
 \end{align}
> $$
> 模型的选择方法为：
> 1. 使用训练集训练 10 个模型，分别得到 10 个 $w$ 和 $b$ 
> 2. 使用 10 个模型分别对交叉验证集计算得出交叉验证误差（代价函数的值）
> 3. 选取代价函数值最小的 $\vec{w}$ 和 $b$
> 4. 用 3 选出的模型对测试集计算得出泛化误差（代价函数）

Training error: ${J_{train}}(\vec{w}, b)=\frac 1 {2m_{train}} \displaystyle \sum_{i=1}^{m_{train}}(f_{\vec{w}, b}(\vec{x}^{(i)}-y^{(i)})^2$
Cross Validation error: ${J_{cv}}(\vec{w}, b)=\frac 1 {2m_{cv} }\displaystyle \sum_{i=1}^{m_{cv}}(f_{\vec{w}, b}(\vec{x}^{(i)}_{cv}-y^{(i)}_{cv})^2$
Test error: ${J_{test}}(\vec{w}, b)=\frac 1 {2m_{test}}\displaystyle \sum_{i=1}^{m_{test}}(f_{\vec{w}, b}(\vec{x}^{(i)}_{test}-y^{(i)}_{test})^2$
## 通过偏差于方差诊断算法

-  $J_{train}$ 偏差很高的情况下，是在训练集上的表现不好，数据欠拟合
-  $J_{cv} >> J_{train}$ 方差很高的情况，是在测试集上的表现差于训练集，过拟合

2. 已经实现的有竞争力的算法的性能
