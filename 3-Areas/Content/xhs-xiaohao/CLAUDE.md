## 2. 内容规则

### 2.1 溯源要求（放宽）

xiaohao 允许 `origin: scratch`（临时灵感），但：
- 涉及具体数字必须有出处（精度要求与大号相同，不得编造）
- 涉及历史事件必须属实
- 投资观点如果和大号冲突，要在 publish 时提醒我

### 2.2 category 允许列表

**比大号多出**：`生活随笔` / `交易日记` / `踩坑` / `人设`

所有大号 category（量化/宏观/商品等）也都可用，但通常会用更口语化的表达。

### 2.3 情绪可控放开

允许：
- 第一人称吐槽
- 情绪化表达（"跌懵了"、"血本无归"）
- 分享自己的失误、怀疑、迷茫
- 生活化场景（通勤、吃饭、深夜复盘）

**禁忌**：
- 攻击他人、踩别的博主
- 放大投资风险（"保赚"、"必涨"等暗示）
- 具体荐股

## 3. 写作结构（小红书图文 200-400 字）

```
强场景开头（30-80 字）
  └─ 画面感 / 情绪钩子 / 人设锚点（"作为一个量化研究员..."）

核心冲突/感悟（100-200 字）
  └─ 可以是故事、顿悟、教训

金句或反思（30-50 字）
  └─ 让人产生"啊对"的感觉

（可选，但强烈推荐）引流钩子（20-30 字）
  └─ "更深的数据我发在大号了" / "想看我怎么做回测？大号更新了"
```

## 5. 引流规则（核心价值）

**drives_to 字段**是小号最重要的 frontmatter。

**什么时候填 drives_to**：
- 本篇小号内容是某篇大号文章的"人设版"
- 本篇末尾有自然引导话术
- 填写对应大号文章的 `[[]]`

**引流话术示例**（Claude 写小号时可自动生成候选）：
- "具体数据我在 [搞钱最重要] 发了张图，感兴趣的来看"
- "量化思路太长这里写不下，我另开一个号专门发这个"
- "这部分背后的逻辑挺复杂，我写在大号了"

不是每篇都要强行引流——**强行引流会掉数据**。一周 2-3 篇带引流足够。
