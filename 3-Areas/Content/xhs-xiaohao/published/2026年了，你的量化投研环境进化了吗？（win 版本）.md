## 🐧 Linux 和 AI 编程工具

为了追求极致的响应速度和命令执行的稳定性，在 Windows 上搭建 **WSL 2（Windows Subsystem for Linux）** 跑 Ubuntu 已经是量化圈的标配了。

### 为什么 AI 编程非 Linux 不可？

- **原生环境兼容性：** Linux的POSIX兼容性会让AI工具的CLI跑得更稳，不会莫名其妙报错。
- **性能损耗极低：** WSL 2在处理海量小文件时，I/O 性能远超原生 Windows。
- **工具链丝滑衔接：** Linux 下的权限管理和环境变量配置会让你省掉无数个“环境报错”的坑。

## 📦 Python 环境管理器

🥇 **uv** / 🥈 **conda**

uv 是 Rust 写的 pip + venv 替代品，安装一个完整的量化环境只需要十几秒，比 pip 快 10-100 倍。项目隔离用`uv venv`，依赖锁定用`uv.lock`，跨机器复现环境直接一条命令。

conda 老牌稳健，处理有 C 扩展的包（比如 ta-lib、cvxpy）兼容性更好，可做为备选。

## 🗄️ 数据库

🥇 **ClickHouse** / 🥈 **DuckDB**

量化数据的本质是时序 + 大规模列式查询，ClickHouse 就是为这个场景生的。

DuckDB 是嵌入式 OLAP 数据库，处理 Parquet 文件和 CSV 的速度极快。适合中小规模因子研究、临时数据探索，和 Polars 配合堪称完美。

## 📊 Python 数据分析 

🥇 **Polars** / 🥈 **Pandas**

Polars 用 Rust 写的，多线程并行、零拷贝内存模型，处理千万行日频数据比 Pandas 快 5-20 倍，内存占用也更低。Lazy API 支持查询优化，写起来像 SQL，逻辑清晰。

Pandas 依然是生态最完整的选择，很多第三方库（backtrader、vectorbt）还是只接受 DataFrame，可作为兼容层保留。

## 🔧 其他

- **Zellij**：Rust 写的终端复用器，比 tmux 配置更直觉，内置浮动窗口和标签页，多任务回测时一屏掌控全局。
- **Tailscale**：基于 WireGuard 的零配置组网工具，家里的台式机、公司笔记本、云服务器，一键打通内网，跑远程 ClickHouse 查询跟本地一样丝滑。
- **Node.js**：前端可视化、本地工具脚本的运行时基础。

## ✍️ 写在最后

这套环境不是一天搭出来的，是踩了无数坑之后逐步收敛的结果。

核心思路其实就一句话：**能用 Rust 生态的，就不用老工具。** 当你的数据量从百万行跑到千万行、从单因子跑到多因子组合回测的时候，工具的性能天花板会直接决定你的研究效率。
