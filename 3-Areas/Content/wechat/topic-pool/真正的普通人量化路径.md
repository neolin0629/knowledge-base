## 三、三阶段路径（refined，有具体步骤）

### S1 概率化思维（2–3 个月）

**核心转变**：从"感觉对不对"→"这件事在什么条件下发生的概率是多少"

**具体要掌握的概念**：

1. **条件概率**（不是算术平均）
   - 案例：某资产每天上涨的基础概率是 60%（噪音）；但在成交量高于均值的日子里，上涨概率是 75%（信号）。75% 才是值得入场的依据，60% 是背景噪音。
   - 实践输出：你能不能写出"我只在 XX 条件同时成立时开仓"的条件清单？

2. **贝叶斯更新**（新信息如何改变你的判断）
   - 核心逻辑：先验信念 × 新证据似然 → 后验信念
   - 实践输出：当一个"对我有利的消息"出来时，你能不能量化说出"我原来认为概率是 P0，看到这个消息后修正为 P1"，而不是直接加仓？

3. **期望值 × 方差**（两者都要，不能只看期望）
   - 关键点：期望值为正但方差极大，足以在"长期"到来之前先把你打出局。
   - 案例：期望每笔赚 ¥200，标准差 ¥5000，本金 ¥20000 → 3连亏可能就爆了。

4. **凯利公式**（科学仓位管理）
   - 实践用法：算出凯利值后**打半折**（半凯利）。两篇原文都明确建议：实战中对胜率的估计总有误差，全仓凯利和全仓梭哈差不多危险。
   - 具体公式：f = (p·b - q) / b，其中 p = 胜率，q = 1-p，b = 赔率

**推荐资料**：
- Blitzstein & Hwang《Introduction to Probability》前 6 章（哈佛免费 PDF，[probabilitybook.net](http://probabilitybook.net/)）
- 配套编程：Python 模拟 10,000 次抛硬币验证大数定律（原文附代码，直接 copy）

**阶段输出**：能写出自己交易的"条件清单"——只在 XX 条件下开仓，并能估算该条件的历史概率

### S2 纪律化执行（3–6 个月）

**核心转变**：从"我赚了多少"→"我赚的里面有多少是 Alpha，多少是 Beta，多少是运气"

**具体要掌握的概念**：

1. **多重比较陷阱**（这是普通人被坑最深的一个点）
   - 数据：如果你让电脑随机生成 1000 个策略跑历史数据，其中纯靠运气就有约 50 个策略会显示"显著盈利"（p < 0.05）。
   - 原文原话：「你写出的前 10 个策略，绝对都是那几只运气好的猴子。」（这是两篇原文都明确写的，不是我猜的。）
   - 对策：Bonferroni 校正（人话版）——如果你测试了 N 个策略，你的显著性门槛不是 0.05，而是 0.05/N。

2. **Alpha 自评**（剔除市场 Beta 后你还剩什么）
   - 线性回归最简单的用法：把你的策略收益 vs 同期沪深 300 / 对标指数，看截距 α 是否显著大于 0。
   - 如果策略今年赚了 15%，但大盘同期涨了 12%，你的 Alpha 只有 3%——还不算风险调整后的。
   - 更进一步（选做）：Fama-French 三因子回归，用 statsmodels 跑，看 alpha 在控制规模、价值因子后是否还在。

3. **金融数据的特殊性**
   - 收益率不服从正态分布（肥尾/Fat Tails）：用 MLE 拟合 t 分布比正态分布更准
   - 收益率有自相关性 + 异方差：用 Newey-West 标准误差修正，否则你的统计检验过于乐观

4. **交易日记标准化**（这是最被低估的工具）
   - 每笔交易记录：进出场理由（条件清单中哪些触发了）+ 预设止损位 + 事后归因（赚了是因为判断对了还是运气好？亏了是判断错了还是执行歪了？）

**推荐资料**：
- Wasserman《All of Statistics》第 1–13 章（CMU 免费 PDF）
- statsmodels 文档（做回归用）
- 配套编程：yfinance 下载真实收益率数据 → 正态检验（100% 会失败）→ MLE 拟合 t 分布

**阶段输出**：
- 一张自己过去 1 年的「盘感 vs Alpha」对照表（能区分运气和能力）
- 一份标准化交易日记模板（自己填得下去的版本，不是那种你永远不会用的复杂表格）

### S3 最小工具化（6–12 个月）

**核心转变**：从"概念清楚但没有工具"→"有一个能跑的完整闭环"

**工具链（只选最小可行集）**：

| 工具 | 用途 | 为什么选这个 |
|---|---|---|
| Python + pandas | 数据处理 | 不上 polars（太早，pandas 够用） |
| Tushare / AkShare | 中国市场免费数据 | 不买 Polygon（$199/月，专业用） |
| yfinance | 美股/ETF 数据 | 免费 |
| vectorbt | 起步回测 | 比 backtrader 更简单，比 NautilusTrader 低一个数量级的复杂度 |
| statsmodels | 统计回归 | Alpha 自评用 |
| matplotlib | 可视化 | 够用 |

**最小闭环长这样**：
```
原始数据（Tushare/AkShare）
    ↓
信号生成（你的条件清单 → Python 判断）
    ↓
回测（vectorbt）
    ↓
统计验证（多重比较校正 + Alpha 回归）
    ↓
模拟盘（不是实盘，至少跑 3 个月）
    ↓
实盘（小仓位）
```

**明确不学清单（节省 12–18 个月）**：
- ❌ Black-Scholes / 伊藤引理 / 随机微积分（除非你做期权做市）
- ❌ C++ / Rust / FPGA（你的优势不在低延迟，跟专业机构拼延迟必输）
- ❌ QuantLib（衍生品定价库，不做机构衍生品用不上）
- ❌ NautilusTrader（生产级系统，你还没到那一步）
- ❌ Shreve《Stochastic Calculus for Finance》（那是专业 Quant 的必读，不是你的必读）
- ❌ MIT 18.06 线性代数全套 + Markowitz 优化（工具层面的 cvxpy 会用就够了）

**推荐资料**：
- Ernest Chan《Quantitative Trading》（2008）——散户向量化入门最接地气的书
- vectorbt 官方文档
- Tushare / AkShare 官方文档

**阶段输出**：一个能跑的最小闭环——数据 → 信号 → 回测（含多重比较修正）→ 模拟盘

## 四、反常识点清单（文章立场，每条要有数据支撑）

1. **普通人入门量化最该先做的不是装 Python，是把过去一年交易记录算一遍 Alpha**
   - 数据：大多数散户"赚钱了"只是大盘 Beta，不是 Alpha

2. **网上 90% 的入门路径是在培养「想去投行的理科生」，不是「让散户少亏点」**
   - 数据：两篇原文明确对标 Jane Street 入门薪 $300K–$500K，这不是你的目标

3. **你的前 10 个策略大概率是多重比较的噪声**
   - 数据：随机生成 1000 个策略，50 个会纯靠运气显示显著盈利（p<0.05）；你最初只测试 10 个，但你做到"看起来不错"才停下来，这本身就是筛选偏差

4. **凯利公式的正确用法是"打半折"**
   - 原文明确写：对胜率的估计总有误差，全仓凯利比全仓梭哈也好不到哪儿去

5. **最被高估的入门难度：随机微积分；最被低估的入门难度：交易纪律**
   - 原文原话（gemchanger）：「Estimation error is the real enemy.」参数估计误差是真正的敌人，数学在参数准确时完美，你永远没有准确的参数

6. **工具已经民主化，判断力没有**
   - 原文原话：「Tools have democratized. Conviction hasn't.」任何人都能 pip install vectorbt，优势在于独特的视角和纪律，不在于更好的工具

**补充：**
### López de Prado Chapter 8 具体论点？
根据书中第 8 章（“测试集过拟合 / Testing Set Overfitting”）以及其他相关章节的内容，作者 Marcos M. López de Prado 针对金融机器学习中极其危险的**多重测试/比较（Multiple Testing / Comparisons）**问题以及**组合清洗交叉验证（Combinatorial Purged Cross-Validation, CPCV）**的应对思路提出了以下核心论点：
1. 多重比较（Multiple Testing）的危险与“虚假策略定理”

- **多重测试下的选择偏差（SBuMT）**：回测并不是一种可控实验，研究人员常常会进行成千上万次的历史模拟（多重测试），然后只挑出表现最好（如夏普比率最高）的策略。这种做法会导致测试集的过拟合，并产生虚假发现（False Discoveries）。
- **第一类错误率呈指数上升**：即使策略的真实期望收益为零，如果研究人员反复进行 $K$ 次独立的测试，只要测试次数足够多，至少出现一次“假阳性”（第一类错误）的族系误差率（Familywise Error Rate, FWER）就会大幅上升。
- **虚假策略定理（False Strategy Theorem）**：该定理指出，在一个毫无预测能力的策略集合中，仅仅由于测试次数的增加，预期能够得到的最大夏普比率（Expected Maximum Sharpe Ratio）也会严格大于零。这意味着表现优异的回测结果可能仅仅是多次尝试带来的统计假象。

2. 应对多重比较的解决方案

为了防止多重测试带来的回测结果虚高，作者在第 8 章提出了几种量化和调整方法：

- **缩水夏普比率（Deflated Sharpe Ratio, DSR）**：这是一种控制了多重测试效应的评估指标。它利用虚假策略定理推导出的预期最大夏普比率作为新的零假设阈值，同时还对收益率序列的偏度（Skewness）、峰度（Kurtosis）以及样本长度进行调整，从而计算出观测到的夏普比率在统计上的真实显著性。
- **估算“有效测试次数”与 Šidàk 修正**：在实际业务中，研究人员测试的多个策略往往具有高度的相关性（并非完全独立的测试）。作者建议先使用聚类算法（如 ONC 算法）对所有尝试过的策略回测序列进行聚类，从而估算出“有效独立测试次数（Effective Number of Trials, $E[K]$）”。在得到这一有效次数后，可以应用 Šidàk 修正公式来调整单次测试的显著性水平，从而有效控制族系误差率（FWER）。

3. 组合清洗交叉验证（Combinatorial Purged Cross-Validation, CPCV）的作用

虽然多重比较问题揭示了测试集过拟合的原因，但通过 **CPCV** 可以在重采样层面主动防御这种过拟合（具体概念在第 1 章及附录 A 中有相关论述）：

- **应对单一测试集的脆弱性**：研究人员很容易在单个测试集上过拟合，但模型很难在成千上万个不同的测试集上同时过拟合。CPCV 通过对训练集和测试集进行组合拆分与重采样来解决这一问题。
- **生成多条独立历史路径**：附录 A 中指出，通过使用类似于“留 k 验证法（leave k out）”的组合交叉验证技术，可以生成大量的历史回测路径。与单一路径的历史回测相比，这种基于大量组合生成的合成测试集更难被过拟合，极大地提高了策略在样本外的稳健性。
