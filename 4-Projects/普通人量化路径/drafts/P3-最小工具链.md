## 正文

任何人都能 `pip install vectorbt`。

优势不在工具，在判断力和纪律。工具已经民主化了，判断力没有。（"Tools have democratized. Conviction hasn't." — Ernest Chan, 2008）

前两篇讲了思维和纪律，这篇讲工具——以及更重要的，哪些你根本不需要学。

**你真正需要的最小工具链**

| 工具 | 用途 | 为什么选这个 |
|---|---|---|
| Python + pandas | 数据处理 | 够用，不需要 polars |
| Tushare / AkShare | A 股免费数据 | 无需付费数据源（$199/月 的 Polygon 是机构用的） |
| yfinance | 美股 / ETF 数据 | 免费 |
| vectorbt | 回测 | 比 backtrader 更简单，比 NautilusTrader 低一个量级的复杂度 |
| statsmodels | 统计验证 | Alpha 回归 + 多重比较校正，上一篇讲的那些 |

**完整闭环长这样**

```
原始数据（Tushare / AkShare）
    ↓
信号生成（条件清单 → Python 判断）
    ↓
回测（vectorbt）
    ↓
统计验证（多重比较校正 + Alpha 回归）
    ↓
模拟盘（至少跑 3 个月再上实盘）
    ↓
实盘（小仓位）
```

**你不需要学这些（每条省 1–3 个月）**

- ❌ **Black-Scholes / 伊藤引理**：期权做市商才用得上。你做的是方向性判断，不是衍生品定价。
- ❌ **C++ / Rust / FPGA**：你的优势是判断力，不是低延迟。和机构拼延迟，永远输。
- ❌ **QuantLib**：机构衍生品定价库，与你的护城河无关。
- ❌ **NautilusTrader**：生产级交易系统，你还没到那一步。上实盘的仓位还不需要这个级别的基础设施。
- ❌ **Shreve《Stochastic Calculus for Finance》**：专业 Quant 的必读。不是你的必读，学了也用不到 5%。
- ❌ **MIT 18.06 线性代数全套**：做因子回归，statsmodels 里的 OLS 够了。真需要优化，cvxpy 调包即可。

这些都是真实存在的高质量内容，只是它们的目标读者不是你。

**结尾**

这条路不性感。没有推导 Black-Scholes 的成就感，没有从 K 线悟道的浪漫——只有一份条件清单、一份交易日记、一个能跑的回测闭环。

但它能让账户活下来。

工具是平权的。判断力，不是。
